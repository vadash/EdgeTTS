# LLM Robustness and Local Assets — Design v2

## 1. Overview

Upgrade the Edge-TTS Web extraction and parsing pipeline by adapting battle-tested patterns from OpenVault. The goal is to maximize compliance and reliability from mid-tier, cost-effective CN models (Qwen, Kimi) and eliminate external CDN dependencies for WebAssembly assets.

**Changes from v1:** Added language mirroring rules for non-English stories, multiple prefill presets (not just one hardcoded), complete CN/EN preamble text from OpenVault, and a non-destructive prompt topology refactor that preserves existing rich rules/examples.

## 2. Core Architecture Changes

### A. The JSON Repair Pipeline (`src/utils/text.ts`)

Instead of directly parsing LLM outputs, all responses will pass through a multi-stage sanitizer before Zod validation.

1. **`stripThinkingTags(text)`**: Regex-based removal of:
   - Paired XML tags: `<think>...</think>`, `<reasoning>...</reasoning>`, `<tool_call>...</tool_call>`, `<search>...</search>` (with optional attributes)
   - Paired bracket tags: `[THINK]...[/THINK]`, `[TOOL_CALL]...[/TOOL_CALL]`
   - Asterisk-wrapped thinking: `*thinks: ...*`
   - Parenthesized thinking: `(thinking: ...)`
   - Orphaned closing tags (opening tag was in assistant prefill): `^.....</think>`

2. **`extractBalancedJSON(text)`**: A bracket-counting algorithm that scans the text and extracts the *last* complete `{}` or `[]` block. Why "last"? LLMs output reasoning and hallucinated `<tool_call>` snippets BEFORE the actual payload. The real JSON is always the last complete block.

3. **Syntax Sanitization**: Regex replacements to fix mid-string concatenations (`"text" + \n "more"`), dangling plus signs before punctuation, and unclosed quotes (odd unescaped `"` count → pad with `"]}]}`).

4. **`jsonrepair` integration**: Fixes trailing commas, missing quotes on keys, and other structural JSON issues. Must be added as a dependency (`npm install jsonrepair`).

5. **`safeParseJSON<T>(text, schema)`**: Orchestrates the above steps, parses the JSON, and validates against the provided Zod schema. **Throws on failure** (unlike OpenVault's null-return pattern) because EdgeTTS wraps all LLM calls in `withRetry` + `RetriableError`.

### B. Prompt Topology & Recency Bias Defeat

Refactor prompt assembly (NOT prompt content) to separate structural components. The existing rich rules, examples, and formatting in `extract.ts`, `merge.ts`, and `assign.ts` are **preserved intact** — only their container structure changes.

#### System Prompt = Preamble + Role + Examples
- **Preamble**: Anti-refusal framing (`SYSTEM_PREAMBLE_CN` or `SYSTEM_PREAMBLE_EN`). Frames the task as a production data pipeline with pre-authorization.
- **Role**: Existing role description from each prompt file.
- **Examples**: Existing few-shot examples from each prompt file.

#### User Prompt = Content + Language Rules + Task Rules + Schema + Trigger
- **Content**: The actual text/characters/paragraphs to process (existing `userTemplate`).
- **Language Rules**: `MIRROR_LANGUAGE_RULES` block — ensures non-English stories get non-English output values.
- **Task Rules**: Existing rules (extraction rules, merge rules, assignment rules).
- **Output Schema**: Existing output format specification.
- **Execution Trigger**: Affirmative instruction at the very end to defeat recency bias.

#### Assistant Prefill (3rd message)
Inject a third message into every API call:
```
{ role: 'assistant', content: '<think>\n...' }
```
Multiple presets available (CN compliance, EN compliance, step-by-step, etc.). Default: `<think>\n` (pure think tag — safest for most models).

### C. Language Rules for Non-English Stories

Many target stories (LitRPG, Web Novels) are in Chinese or Russian. Without explicit language rules, mid-tier models default to English output values even when the source text is non-English.

```
<language_rules>
OUTPUT LANGUAGE PROTOCOL:
• KEYS = ENGLISH ONLY. Never translate JSON keys.
• VALUES = SAME LANGUAGE AS SOURCE TEXT.
• NAMES = EXACT ORIGINAL SCRIPT. Never transliterate.
• THINK BLOCKS = ENGLISH ONLY.
</language_rules>
```

This block is always injected into the user prompt constraints.

### D. Local Asset Bundling (WASM)

Eliminate network flakiness caused by `unpkg` and `jsdelivr`.

- **Webpack Config**: Add `CopyWebpackPlugin` rules to copy `@ffmpeg/core/dist/umd/ffmpeg-core.js` and `ffmpeg-core.wasm` into the output directory.
- **`FFmpegService.ts`**: Remove the `tryLoadFromMirrors` CDN loop. Update `.load()` to fetch the core and WASM files via relative local paths (e.g., `./ffmpeg-core.js`).
- **`src/config/index.ts`**: Remove `FFmpegCDNMirror` interface, `FFmpegConfig` interface, and the `ffmpeg` block from `defaultConfig`/`AppConfig`.
